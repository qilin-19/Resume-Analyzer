# 简历评分标准参考

> 这份资料让 LLM 按统一标准打分，而不是凭直觉。
> 你可以根据自己偏好调整分数段位和权重。

---

## 技能匹配度（满分 50 分）

| 命中率     | 得分范围  | 说明                           |
|-----------|----------|-------------------------------|
| >= 80%    | 45-50 分 | 技能高度匹配，基本满足 JD 全部要求  |
| 60-79%    | 35-44 分 | 大部分匹配，有少量缺口需要补       |
| 40-59%    | 25-34 分 | 半数匹配，需要针对性提升           |
| 20-39%    | 10-24 分 | 匹配度低，可能需要重新定位岗位方向   |
| < 20%     | 0-9 分   | 严重不匹配，建议大幅调整技能方向    |

---

## 项目经历质量（满分 50 分）

| 指标              | 得分     | 说明                         |
|------------------|---------|-----------------------------|
| 3+ 个项目 + 量化数据 | 40-50 分 | 经历丰富，成果可衡量            |
| 2-3 个项目 + 部分量化 | 30-39 分 | 有一定积累，数据不够充分         |
| 1-2 个项目 + 少量量化 | 20-29 分 | 经历偏少，需要补充实战           |
| 1-2 个项目 + 无量   | 10-19 分 | 有经历但缺乏数据支撑            |
| 无明确项目          | 0-9 分   | 简历需要重新组织，突出项目经验     |

---

## 加分项（硬指标之外的亮点）

- **开源贡献**：有 GitHub PR 被合并 / 维护开源项目 → +2~5 分
- **技术博客**：有持续输出（掘金、知乎、个人博客）→ +2~5 分
- **社区影响力**：GitHub stars > 100 / 技术演讲经历 → +2~5 分
- **跨领域能力**：同时具备前后端 / 懂产品 / 懂设计 → +2~5 分
- **英语能力**：能阅读英文文档 / 英文简历 → +1~3 分

> 加分项不计入基础 100 分，单独计算后作为"额外亮点"展示。

---

## 改进建议分级

### 🔴 高优先（直接影响投递结果）
- JD 要求的必备技能缺失
- 项目经历缺少量化数据
- 简历中没有关键技术关键词

### 🟡 中优先（提升竞争力）
- 有技能但简历中没体现
- 项目数量偏少（< 3 个）
- 缺少行业热门技术

### 🔵 低优先（锦上添花）
- 可以补充软技能描述
- 可以优化简历排版和措辞
- 可以添加个人技术博客链接
